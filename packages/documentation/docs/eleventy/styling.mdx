## Overriding and customization

You can pass custom classes to the configuration:

```js
const footnotes = require("eleventy-plugin-footnotes");

module.exports = function (eleventyConfig) {
  eleventyConfig.addPlugin(footnotes, {
    classes: {
      container: "my-container",
      title: "my-title",
      ref: "my-ref",
      list: "my-list",
      listItem: "my-listItem",
      backLink: "my-backLink",
    },
  });
};
```

:::tip

You do not _have_ to use these for styling. You can also leverage the fact that the library renders `role` attributes and stick to selectors like `[role="doc-noteref"]`. As you prefer.

:::
